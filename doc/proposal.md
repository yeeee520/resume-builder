# Proposal

## Goal

构建一个**本地简历拼图构建器**——用 React 组件化的方式，让用户可以像搭积木一样自由组合简历模块，实时预览，一键导出 PDF，并支持从已有的 PDF/Word 简历中智能识别内容自动生成拼图块。

## Non-Goals

- 不做后端/云存储（纯本地）
- 不做多语言国际化（仅中文 UI）
- 不做移动端适配（桌面优先）
- 不用 AI / LLM 做内容识别（纯启发式规则，隐私安全）

## Users And Runtime Context

- 单用户，本地使用
- Windows 11 桌面环境
- 浏览器中运行，npm run dev 启动，提供一个 .bat 一键启动脚本

## Functional Requirements

### 核心编辑器
- **左侧组件库面板**：展示可用拼图块类型（标题、正文、分割线、技能条、时间段、头像区等），点击或拖入画布
- **中央画布**：A4 比例预览区，自由拖拽排序，实时渲染
- **右侧属性面板**：选中某块后，编辑其内容和样式（字体、颜色、大小、间距等）
- **撤销/重做**：Ctrl+Z / Ctrl+Y

### 智能导入（新增）
- **导入 PDF**：上传 PDF 简历 → 提取文字 → 按内容类型智能分类 → 预览确认 → 自动生成拼图块
- **导入 Word (.docx)**：上传 .docx 简历 → 提取文字 + 保留粗体/标题语义 → 智能分类 → 预览确认 → 生成拼图块
- **智能分类规则**（纯本地正则 + 启发式，无需 AI）：
  - 姓名：首行 / 字号最大的文本
  - 联系方式：匹配手机号、邮箱、GitHub、LinkedIn 等正则
  - 教育/工作经历：匹配日期范围 + 标题 + 描述的模式
  - 技能：匹配短词列表、百分比（如 "Python 90%"）
  - 标题段落：识别章节标题（短文本、单独成行、加粗）
  - 正文：其余大段文字
- **预览确认界面**：弹窗展示识别结果（类型、内容摘要），用户可修改分类或忽略某条 → 确认后自动插入画布

### 拼图块类型（可扩展）
- 标题块（H1/H2/H3 可选，加粗/斜体/下划线）
- 正文块（多行文本，支持简单 Markdown）
- 分割线块（粗细、颜色、样式）
- 技能条块（名称 + 进度百分比）
- 时间段块（时间段 + 标题 + 描述，适用于教育/工作经历）
- 头像块（图片上传 + 裁剪为圆形或方形）
- 标签组块（多个标签，可选颜色）
- 联系方式块（电话、邮箱、GitHub 等 K/V 对）
- 间距块（可拖拽高度的空白区域）

### 数据管理
- **localStorage 自动保存**：每次修改自动保存，刷新不丢失
- **多份简历**：可创建、切换、删除多份简历
- **JSON 导出/导入**：备份和迁移（导入 .json 文件）

### 主题切换
- 经典黑白
- 现代蓝色
- 简约灰白
- 后续可扩展

### 导出
- 一键导出 PDF（按钮触发）
- 导出时自动隐藏编辑器 UI，仅保留简历内容

### 一键启动
- 提供 `启动简历编辑器.bat`，双击打开浏览器

## Inputs And Outputs

- 输入：用户通过 UI 编辑的内容、PDF 文件（.pdf）、Word 文件（.docx）、JSON 配置文件（.json）
- 输出：JSON 配置文件、PDF 文件

## Dependencies And Constraints

- React 18 + TypeScript
- Vite 构建
- @dnd-kit 拖拽库（动画流畅）
- html2canvas + jsPDF 导出
- pdfjs-dist PDF 解析
- mammoth.js Word (.docx) 解析
- Tailwind CSS 样式
- 不需要数据库，不需要后端，不需要 AI/LLM

## Success Criteria

- 能在 2 秒内启动并打开浏览器
- 拖拽动画 ≥ 60fps
- 添加/编辑/删除/排序任意拼图块流畅无卡顿
- PDF 导出与画布预览一致
- 多份简历切换无缝
- 刷新页面后数据完好
- 导入标准格式的 PDF/Word 简历，识别准确率 ≥ 80%（主要字段正确识别）

## Acceptance Checks

- 双击 .bat → 浏览器自动打开 → 编辑器界面完整展示
- 从组件库拖入 5 个不同类型的块 → 全部正确渲染
- 编辑某块内容 → 实时更新
- 切换主题 → 全局样式变化
- 导出 PDF → 打开后内容与预览一致
- 关闭浏览器重新打开 → 上次编辑的内容恢复
- 导出 JSON → 新建简历 → 导入 JSON → 内容恢复
- 导入一份标准 PDF 简历 → 识别出姓名、联系方式、经历 → 预览确认 → 拼图块正确生成
- 导入一份标准 Word 简历 → 识别出章节标题、时间线、技能 → 拼图块正确生成
- 预览界面可修正识别结果（改类型、删条目）

## Assumptions

- 用户已安装 Node.js 18+
- npm install 首次执行即可
- 浏览器版本较新（Chrome/Edge 最近 2 年）
- 导入的简历排版规范（标准 A4，非扫描图片 PDF）
